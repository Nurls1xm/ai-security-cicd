# Сценарий демонстрации проекта AI Security Analysis for CI/CD

## Информация о проекте

**Название:** AI Security Analysis for CI/CD  
**Технологии:** n8n, LangChain, Google Gemini AI, Jenkins, Docker  
**Цель:** Автоматический анализ безопасности кода с помощью AI перед deployment  

---

## Структура видео (общая длительность: 10-12 минут)

### 1. Введение (1 минута)

**Что показать:**
- Титульный слайд с названием проекта
- Краткое описание проблемы

**Что сказать:**
```
"Здравствуйте! Сегодня я представлю проект 'AI Security Analysis for CI/CD'.

Проблема: При разработке ПО разработчики могут случайно добавить опасный код, 
который создает уязвимости безопасности. Традиционные сканеры находят только 
известные уязвимости, но не понимают контекст.

Решение: Мой проект использует искусственный интеллект (Google Gemini) для 
анализа кода, а OPAL (Open Policy Administration Layer) для управления 
политиками безопасности и принятия финальных решений о deployment."
```

### 2. Архитектура системы (1.5 минуты)

**Что показать:**
- Откройте README.md и покажите диаграмму архитектуры
- Или нарисуйте схему на доске/экране

**Схема:**
```
Developer → GitHub → Jenkins → n8n → LangChain → Gemini AI
                        ↓                           ↓
                   Scan tests/              Analyze & Decide
                        ↓                           ↓
                   Find patterns              Risk Assessment
                        ↓                           ↓
                   Send to n8n ←──────────────────┘
                        ↓
                   OPAL Policy Check
                        ↓
                   Final Decision
                        ↓
                   Deploy or Block
```

**Что сказать:**
```
"Архитектура проекта состоит из нескольких компонентов:

1. GitHub - хранит код проекта
2. Jenkins - CI/CD система, которая автоматически запускается при каждом push
3. Security Scanner - сканирует папку tests/ на опасные паттерны: eval(), exec(), os.system()
4. n8n - workflow automation, связывает Jenkins с AI
5. LangChain + Gemini AI - анализирует результаты сканирования и дает рекомендацию
6. OPAL - проверяет политики безопасности и принимает финальное решение
7. Результат - либо разрешает deployment (YES), либо блокирует (NO)

Ключевое отличие: AI дает рекомендацию, а OPAL применяет политики организации 
и принимает финальное решение. Это позволяет гибко управлять правилами 
безопасности без изменения кода.

Все компоненты работают в Docker контейнерах."
```

```
Посмотрим на основные правила в security.rego:

1. allow_deployment - разрешает deployment если:
   - LOW risk + confidence ≥ 0.8
   - MEDIUM risk + confidence ≥ 0.9 + нет критических проблем
   
2. deny_deployment - блокирует deployment если:
   - CRITICAL или HIGH risk
   - Найдены опасные паттерны
   
3. deployment_decision - принимает финальное решение на основе правил

Преимущество OPAL: можно изменить политики без изменения кода - 
просто обновляем .rego файл и делаем git push."
```

## Часто задаваемые вопросы (для Q&A)

**Q: Почему сканируется только папка tests/?**  
A: Для демонстрации. В production можно сканировать весь репозиторий.

**Q: Какие еще паттерны можно добавить?**  
A: SQL injection, XSS, hardcoded secrets, unsafe deserialization и т.д.

**Q: Можно ли использовать другие AI модели?**  
A: Да, n8n поддерживает OpenAI, Claude, и другие через LangChain.

**Q: Зачем нужен OPAL если есть AI?**  
A: AI дает рекомендацию, OPAL применяет политики организации. Это разделяет 
   анализ и принятие решений, позволяя гибко управлять правилами без изменения кода.

**Q: Можно ли изменить политики OPAL без перезапуска?**  
A: Да, OPAL автоматически подтягивает изменения из Git репозитория.

**Q: Как масштабировать на большие проекты?**  
A: Использовать параллельное сканирование, кеширование результатов, 
   и более мощные AI модели.

**Q: Сколько стоит запуск?**  
A: Gemini API имеет бесплатный tier. Для production нужен платный план.
   OPAL и OPA - open source и бесплатны.
